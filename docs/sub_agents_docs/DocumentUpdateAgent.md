# Sub-Agent Reference Template (공통용, 모듈화 강조)

## 목적

* 사용자가 제공한 데이터를 기반으로 프로젝트 문서를 자동 생성/업데이트
* 앱/웹 모두에서 재사용 가능하도록 공통 구조 제공
* 핵심 기능, UX, 데이터, 법적/운영 요소 반영
* **모든 작업은 모듈화 가능하게 설계**

## 작업 프로세스

### 1. 내용 분석

* 주제 파악 (기능/스펙/정책 등)
* 기존 문서와 연관성 판단
* 개인정보, 보안, 법적 고려 사항 확인
* **모듈 단위로 분석/처리 가능하게 분리**

### 2. docs/planning/ 문서 처리

* 관련 문서 찾기 → 있으면 Edit, 없으면 신규 생성
* 구조: 목적 → 구조 → UX → 데이터 → 구현 → 법적/운영 포인트
* **각 항목별로 독립 모듈로 작성**

### 3. docs/sub_agents_docs/ 문서 처리

* 관련 에이전트 문서 찾기 → 있으면 Edit, 없으면 신규 생성
* **모듈 단위로 병렬 처리 가능하도록 설계**
* **문서는 너무 길지 않게 요약하고, 반복 내용은 한 번만 포함**

**서브에이전트 문서 구조 (요약)**

```markdown
# [기능명] Agent Reference

## [Planning 문서명] (파일명.md)
[해당 planning 문서의 전체 내용]

---

## AI 작업 지침
- 프로젝트 기준 문서 자동 생성/업데이트
- 개인정보 비식별, 데이터 안전, 법적 준수
- **모듈화 구조 우선 적용, 병렬 처리 가능**
- 사건/기능 요약은 유형화/표준화된 표현 사용
- 앱/웹 특성에 따라 UI/기능 세부 조정 가능
- **문서는 간결하게, 반복 내용은 한 번만 포함**
```

## 주요 원칙 (공통)

* **일관성**: 문서 구조/톤 유지
* **명확성**: 애매한 표현 금지, 개인정보 비식별
* **재사용성**: 앱/웹 공통 기능 기준
* **확장성**: UI/UX 세부 조정 가능
* **모듈화**: 모든 작업 독립 단위, 병렬 처리 가능

## 완료 보고 형식

```
✅ 문서 업데이트 완료
📝 업데이트: docs/planning/[파일명].md
📝 업데이트: docs/sub_agents_docs/[파일명].md
(또는)
🆕 생성: docs/planning/[파일명].md
🆕 생성: docs/sub_agents_docs/[파일명].md
```
