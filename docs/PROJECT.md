# 프로젝트 개요

## 서비스명
밥모아 (Babmoa) - 회식 장소 결정 + 주차 데이터 수집 플랫폼

## 한 줄 정의
회식 장소 결정 웹 도구를 통해 **방문 기반 주차 데이터를 자연스럽게 수집**하고, 이를 **B2B 상품으로 전환**하여 수익화하는 플랫폼

---

## 핵심 전략

| 구분 | 설명 |
|------|------|
| 표면 제품 | 회식 장소 투표 도구 |
| 실제 가치 | 방문 기반 주차 데이터 축적 |
| 차별점 | "주차 가능 여부"가 아닌 **"주차 성공 확률"** 제공 |

> **앱/웹은 미끼, 데이터가 본체**

---

## 비즈니스 모델

### B2B 데이터 판매
| 대상 | 활용 |
|------|------|
| 프랜차이즈/외식 브랜드 | 신규 매장 주변 주차 환경 평가 |
| 지도/내비 서비스 | 실제 체감 주차 성공률 제공 |
| 상권 분석/부동산 | 주차 난이도 기반 입점 후보지 평가 |
| 배달/픽업 서비스 | 주차 성공률 반영 배달 효율 최적화 |

### 데이터 제공 방식
- **API**: 실시간 데이터 요청
- **대시보드/리포트**: 웹, PDF, CSV
- **구독형**: 월/분기 단위 업데이트

---

## MVP 범위

### 포함 ✅
- 투표방 생성/공유
- 팀원 투표
- 투표 결과 확인
- 주차 경험 1클릭 기록
- 데이터 수집/집계

### 제외 ❌
- 추천 알고리즘
- 별점/리뷰 작성
- 광고 삽입
- 전국 단위 확장

---

## 초기 KPI

| 지표 | 설명 |
|------|------|
| 투표 생성 수 | 서비스 활성도 |
| 투표 참여자 수 | 바이럴 효과 |
| 투표→기록 전환율 | 데이터 수집 루프 작동 여부 |
| 식당당 주차 기록 수 | 데이터 신뢰도 |
| 반복 사용률 | 제품 가치 검증 |

> 초기 성공 기준: 트래픽이 아닌 **데이터 수집 루프 작동 여부**
